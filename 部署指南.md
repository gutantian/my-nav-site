## 快速操作回顾

### 1. Cloudflare Pages 项目创建

1. 打开 [dash.cloudflare.com](https://dash.cloudflare.com)
2. 左侧菜单 → **Workers & Pages**
3. 点 **Create application** → 选择 **Pages** tab → **Connect to Git**
4. 授权 GitHub，选择仓库 `gutantian/my-nav-site`
5. **Build command / Output directory** → 全部**留空**
6. 点 **Save and Deploy**，等待约 1 分钟

### 2. 绑定自定义域名 `gutantianred.dpdns.org`

> 由于该域名使用 dpdns.org 动态 DNS 服务，无法直接在 Cloudflare Pages 中激活，需通过 Workers 路由方式处理。

#### 方案一：使用 Workers 路由（推荐）

1. 打开 [dash.cloudflare.com](https://dash.cloudflare.com)
2. 左侧菜单 → **Workers & Pages**
3. 进入项目 `my-nav-site`，点击右侧 **View details**
4. 点击 **Triggers** 标签页
5. 找到 **Custom Domains** → **Add Custom Domain**
6. 输入 `gutantianred.dpdns.org`
7. 按提示完成 DNS 验证

> 若提示域名不在 Cloudflare 管理，需前往 DNS 服务商（dpdns.org）添加 CNAME 记录：
> - **类型：** CNAME
> - **名称：** gutantianred
> - **目标：** `my-nav-site.tan34854.workers.dev`
> - **TTL：** 600（10分钟）

#### 方案二：通过 DNS 页面添加

1. 左侧菜单 → ** Websites & Domains** 或 **DNS**
2. 找到 `gutantianred.dpdns.org` 的管理入口
3. 添加 CNAME 记录后，回到 Pages 项目完成激活

## 常见问题

| 问题 | 解决方案 |
|------|----------|
| 域名无法激活 | 确认 DNS CNAME 已正确配置，等待 5~10 分钟后再试 |
| 部署失败 | 检查 GitHub 仓库是否 Public，以及 Build command 是否留空 |
| SSL 证书报错 | Cloudflare 会自动申请证书，等待 24~48 小时自动生效 |
